About TembeaAfrica

"Tembea" means "explore" in Swahili. We believe every traveler deserves an intelligent, multilingual companion that understands Africa's unique context, culture, and languages.

Our Mission

Making Travel Accessible

To make African travel accessible, safe, and enriching for everyone — powered by AI that understands local context, languages, and culture. We bridge the gap between technology and the rich diversity of the African continent, ensuring no traveler is left behind regardless of the language they speak or the device they use.

Our Vision

A Connected Africa

A connected Africa where every traveler — regardless of language, literacy, or budget — can explore with confidence and discover the continent's extraordinary beauty, heritage, and hospitality. We envision a future where AI serves as a bridge between cultures, making the wonders of Africa accessible to the world.

The Challenge We Solve

The Problem

  • Language barriers prevent tourists from communicating in unfamiliar countries, causing them to miss authentic cultural experiences and local insights
  • No real-time safety data — travelers cannot easily check security ratings, air quality indices, or traffic conditions before venturing out
  • Fragmented tools — weather, flights, hotels, currency conversion, and maps are scattered across dozens of separate applications
  • AI hallucination — traditional chatbots fabricate outdated prices, incorrect weather forecasts, and unreliable safety information
  • Accessibility gap — many travelers need voice-based interaction, but most existing tools are text-only and English-centric

Our Solution

  • One intelligent agent with 11 real-time tools that never guesses — every answer is grounded in verified API data
  • 15+ languages including Kinyarwanda, Swahili, French, Arabic, Chinese, Spanish, and more
  • Voice I/O on every tab — speak in any language, hear responses with stop and pause control
  • 6 interface tabs — Chat, Camera Scanner, PDF Documents, Google Maps, Flights, and Bookings
  • Real booking links — Booking.com, Google Hotels, TripAdvisor, and Google Flights integration for actual reservations
  • YOLO camera scanner — identify landmarks, animals, and objects instantly using computer vision

How It Works

TembeaAfrica uses the ReAct (Reason + Act) pattern. The AI reasons about your question, decides which tools to call, executes them in parallel, and synthesizes a grounded answer with real data.

You Ask

Type a question, speak into the microphone, scan with your camera, or upload a PDF travel guide.

Llama 3.3 70B (70 billion parameters) analyzes your request and decides which tools are needed.

4

Response built from live data, with Google Maps links, booking options, charts, and voice output.

Technology Stack

Built on cutting-edge AI research — Transformer attention mechanisms, convolutional neural networks, and speech processing pipelines

Large Language Model

Llama 3.3 70B via Groq inference — 70 billion parameters with self-attention mechanism. Uses scaled dot-product attention for contextual understanding and native JSON function calling for tool selection.

Computer Vision

YOLOv8n — 3.2 million parameters detecting 80 object classes in real-time. Anchor-free single-stage architecture with Intersection over Union (IoU) scoring and a confidence threshold of 0.3.

Speech Recognition

Whisper Large V3 Turbo via Groq — encoder-decoder transformer converting audio to 80-channel log-Mel spectrograms. Achieves 216x real-time transcription speed across 15+ languages.

Text-to-Speech

Browser SpeechSynthesis API providing native device TTS in 15+ languages. Includes stop and pause control. Works offline after initial page load with no additional API calls required.

Agent Architecture

ReAct (Reason + Act) loop: the LLM reasons about the query, decides which tools to invoke, executes them, observes results, and synthesizes the final answer. Maximum 5 iterations with 3-layer error recovery.

Infrastructure

Flask and Gunicorn backend, single-page HTML/CSS/JS frontend, Docker containerization, deployed on HuggingFace Spaces with 16GB RAM. 17 API keys with automatic rotation providing 1.7M tokens per day.

Safety and Security

We take safety seriously. Our system identifies and mitigates 7 distinct threat categories to protect both users and infrastructure.

API Key Security

All keys stored as encrypted HuggingFace Repository Secrets. Never committed to source code. Colab environments use getpass for hidden input. Keys are rotated automatically on rate limits.

No Arbitrary Code Execution

No eval() or exec() calls anywhere in the codebase. Currency conversion uses API calls rather than expression evaluation. PDF processing uses text extraction only, never executing embedded content.

Anti-Hallucination Design

Camera scanner prompt enforces: "Describe ONLY what is in the image. Do NOT assume location." The agent must use tools for all factual data rather than relying on parametric memory.

Three-Layer Error Recovery

Layer 1: Parse malformed tool calls via regex from failed generation XML. Layer 2: Rotate through all 17 API keys on rate limits. Layer 3: Fall back to LLM knowledge with clear disclaimer.

Clarification Before Action

For trip planning queries, the agent asks for budget, duration, and interests before calling tools. This prevents wasteful API calls and ensures recommendations match the traveler's actual needs.

No Persistent Data Storage

Conversations exist in server memory only and are deleted on session reset. No personal data is collected, no tracking cookies are used, and no user information is stored permanently.
